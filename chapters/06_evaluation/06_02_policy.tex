\section{評価方針}

本研究の目的は，
\textbf{LLM と OpenAPI Generator を組み合わせることで，
マニュアルとデバイスマップから整合性の取れた API 仕様とサーバコードを自動生成できるか}
を検証することである。

したがって，本研究で重要なのは，
「生成された OpenAPI 仕様・flow.json・Flask サーバコードが矛盾なく構成されているか」である。

一方，生成された API を用いて家電が実際に動くかどうかは，
SwitchBot API のクラウド側の応答や Bot 実機の状態に依存し，
本研究の手法自体の成否とは独立である。

以上より，本章では
「仕様生成の正確性・整合性・一貫性・起動性・処理時間・トークン使用量」
を中心に評価を行う。
