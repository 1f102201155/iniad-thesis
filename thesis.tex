\begin{abstract}%%%%%%%%%%%%% 概要 %%%%%%%%
近年，スマートホームの普及に伴い，多種多様な家電製品をネットワーク経由で制御したいという需要が高まっている．
しかし，通信機能を持たないレガシー家電や，標準規格に非対応の独自仕様を持つ機器を IoT システムに統合するためには，
開発者が取扱説明書を精読し，仕様を理解した上で個別にドライバを実装する必要があり，その高い専門性と開発コストが障壁となっていた。

本研究では，大規模言語モデル (LLM) と OpenAPI Generator を組み合わせることで，
家電のマニュアルから Web API 仕様および制御コードを全自動生成するシステム「Auto-IoT」を提案・実装した。
本システムは，LLM による意味抽出と，アルゴリズムによる決定論的なコード生成を組み合わせた「ハイブリッド生成アプローチ」を採用している。
具体的には，マニュアルから抽出した操作手順を一度中間表現 (Intermediate Representation; IR) に構造化することで LLM の幻覚 (Hallucination) を抑制し、
生成された API スタブに対して「マーカーベース注入 (Marker-based Injection)」手法を用いて物理デバイス（SwitchBot Bot）の制御ロジックを安全に適用する。

評価実験の結果，高性能モデル (GPT-4o) を用いた場合に 93.3\% の生成成功率を達成し，
マニュアルの入力から実機の物理操作に至る一連のプロセスが，人手による修正なしに完結できることを実証した．
本手法により，高度なドメイン知識を持たない開発者であっても迅速に IoT システムを構築可能となり，
IoT サービス開発におけるプロトタイピングのコストを大幅に削減できることを示した。
\end{abstract}
